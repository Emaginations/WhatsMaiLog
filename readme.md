# 日志小窥 (WhatsMaiLog)

## 简介

一个简单而强大的日志查看插件，让麦麦能窥见自己的log。当用户发送特定触发词时，插件会自动获取最近一段时间的日志，并通过大语言模型进行简要总结。如果有报错会额外告警。

主项目：[MaiBot](https://github.com/Mai-with-u/MaiBot)

## 配置

有人性化的WebUI设计，请在WebUI插件市场安装并在管理页面进行配置：

<br>

### 插件设置
- **开关**：启用/禁用插件。
- **配置版本**：当前配置版本号。
- **触发词**：用户发送这些消息时触发日志查看，可使用`{bot_name}`占位符自动替换为机器人名称（默认：”发生什么了{bot_name}”和”是不是哪里出问题了”）。
- **查询范围（分钟）**：获取多久之前的日志（1-120分钟）。
- **最大日志行数**：最多获取的日志行数（10-500行）。
- **日志文件路径**：MaiBot日志文件的路径（默认 `logs/maibot.log`）。
- **命令仅限WebUI**：是否只有WebUI可以触发 `/whatsmailog` 命令。

<br>

### 大模型设置（独立提供商）
- **启用LLM总结**：是否使用大模型进行日志总结。**注意：启用后插件会将日志内容发送到你配置的外部 LLM API（默认 DeepSeek），请确保你接受这一行为。**
- **API密钥**：DeepSeek API密钥（WebUI中显示为 `********`，可手动点击显示明文）。
- **API地址**：OpenAI兼容的API端点（默认 `https://api.deepseek.com`）。
- **模型名称**：要调用的模型名称（默认 `deepseek-chat`）。
- **温度**：控制生成的随机性（0-2，步进0.1，默认0.7）。

## 命令

| 命令 | 说明 |
|------|------|
| `发生什么了{bot_name}` | 查看最近N分钟的日志总结，如有报错则告警 |
| `是不是哪里出问题了` | 同上 |
| `/whatsmailog` | 手动触发日志查看（受”命令仅限WebUI”开关控制） |
| `/llmtest` | 测试LLM提供商连接是否正常，发送测试消息并显示回复 |
| `/logpath` | 查看当前配置的日志文件路径和状态 |

> **注意**：所有命令均受”命令仅限WebUI”设置影响。开启后，非 WebUI 平台触发命令时插件仅记录日志，不会发送任何消息。

## 工作原理

1. **触发时机**：当用户发送配置的触发词（如 `发生什么了{bot_name}`）或命令（如 `/whatsmailog`）时触发。
2. **日志获取**：插件根据配置的时间范围（默认10分钟）和日志文件路径（默认 `logs/maibot.log`），读取最近的日志内容。支持的时间格式为 `[YYYY-MM-DD HH:MM:SS]`。
3. **LLM总结**：若启用了LLM，插件调用用户自行配置的独立LLM提供商（OpenAI 兼容 API，默认指向 DeepSeek），对日志进行简要总结（150字以内）。
4. **错误检测**：如果日志中包含 `ERROR` 或 `CRITICAL` 字样，回复中增加 `⚠️` 告警标记。
5. **日志片段**：在消息末尾附加”最新日志片段”（最近3行），帮助快速了解当前状态。
6. **时间戳**：在报告末尾添加生成时间，方便追溯。
7. **日志记录**：所有操作均通过 `self.ctx.logger` 记录详细日志到主进程，方便调试。

## 在WebUI中测试

1. 在WebUI建立聊天
2. 发送 `发生什么了麦麦` 或 `/whatsmailog`
3. 如果看到回复消息包含日志总结和 `[看看日志]` 标记，则说明测试成功
4. 发送 `/llmtest` 可测试 LLM 提供商连接
5. 发送 `/logpath` 可查看当前日志文件路径

## 注意事项

- 请确保 `日志文件路径` 正确指向 MaiBot 的日志文件（默认 `logs/maibot.log`）。
- API密钥在WebUI中默认脱敏显示，可手动切换明文，建议保管好密钥。
- 如果LLM调用失败，插件会自动回退到不带总结的纯日志片段展示。
- 日志文件的时间解析假设格式为 `[YYYY-MM-DD HH:MM:SS]`，如果实际格式不同，可修改 `_get_logs` 方法中的解析逻辑。
- 插件加载时会在主进程日志中输出 `[日志小窥] 插件已启动`，可通过此确认插件正常加载。
- 启用 LLM 生成功能会将日志内容发送至你所配置的外部 API（如 DeepSeek），请确认你接受这一隐私风险。
