# 日志小窥

## 简介

这是一个简单的日志查看插件，让麦麦能窥见自己的log。当用户发送特定触发词时，插件会自动获取最近一段时间的日志，并通过大语言模型进行简要总结。如果有报错会额外告警。

主项目：[MaiBot](https://github.com/Mai-with-u/MaiBot)

## 配置

在WebUI插件管理页面进行配置：

### 插件设置
- **开关**：启用/禁用插件。
- **触发词**：用户发送这些消息时触发日志查看，可使用`{bot_name}`占位符自动替换为机器人名称（默认：“发生什么了{bot_name}”和“是不是哪里出问题了”）。
- **查询范围（分钟）**：获取多久之前的日志（1-60分钟）。
- **最大日志行数**：最多获取的日志行数（10-200行）。
- **日志文件路径**：MaiBot日志文件的路径（默认 `logs/maibot.log`）。

### 大模型设置
- **启用LLM总结**：是否使用大模型进行日志总结。
- **API密钥**：DeepSeek API密钥（WebUI中显示为 `********`，可手动点击显示明文）。
- **API地址**：DeepSeek API地址（默认 `https://api.deepseek.com`）。
- **模型名称**：使用的模型名称（默认 `deepseek-chat`）。
- **温度**：控制生成的随机性（0-2，步进0.1，默认0.7）。

## 命令

| 触发词                     | 说明 |
|---------------------------|------|
| `发生什么了{bot_name}`     | 查看最近N分钟的日志总结，如有报错则告警 |
| `是不是哪里出问题了`       | 同上 |

## 工作原理

1. 用户发送触发词（可自定义）。
2. 插件根据配置的时间范围和日志文件路径，读取最近的日志内容。
3. 调用 DeepSeek API（或兼容的OpenAI API）对日志进行简要总结（200字以内）。
4. 如果日志中包含 `ERROR` 或 `CRITICAL` 字样，回复中增加 `⚠️` 告警。
5. 在消息末尾附加“最新日志片段”（最近3行），并附上 `[看看日志]` 文字提示。
6. 所有操作均通过 `self.ctx.logger` 记录详细日志到主进程。

## 注意事项

- 请确保 `日志文件路径` 正确指向 MaiBot 的日志文件（默认 `logs/maibot.log`）。
- API密钥在WebUI中默认脱敏显示，可手动切换明文，建议保管好密钥。
- 如果LLM调用失败，插件会自动回退到不带总结的纯日志片段展示。
- 日志文件的时间解析假设格式为 `[YYYY-MM-DD HH:MM:SS]`，如果实际格式不同，可修改 `_get_logs` 方法中的解析逻辑。
