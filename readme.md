# 日志小窥 (WhatsMaiLog)

## 简介

一个简单而强大的日志查看插件，让麦麦能窥见自己的log。当用户发送特定触发词时，插件会自动获取最近一段时间的日志和 Maisaka 推理输出，并通过大语言模型进行简要总结。如果有报错会额外告警。

主项目：[MaiBot](https://github.com/Mai-with-u/MaiBot)

## 配置

有人性化的WebUI设计，请在WebUI插件市场安装并在管理页面进行配置：

<br>

### 插件设置
- **开关**：启用/禁用插件。
- **配置版本**：当前配置版本号。
- **触发词**：用户发送这些消息时触发日志查看，可使用`{bot_name}`占位符自动替换为机器人名称（默认："发生什么了{bot_name}"和"是不是哪里出问题了"）。
- **查询范围（分钟）**：获取多久之前的日志（1-120分钟）。
- **最大日志行数**：最多获取的日志行数（10-500行）。
- **日志文件路径**：MaiBot日志文件的路径（默认 `logs/` 目录，自动查找 JSONL 日志和 maisaka_prompt 输出）。
- **命令仅限WebUI**：是否只有WebUI可以触发 `/xx` 形式命令。
- **查你后台触发词**：发送此关键词可提取当前聊天流的 Maisaka 后台推理输出（默认"查你后台"）。

<br>

### 大模型设置（独立提供商）
- **启用LLM总结**：是否使用大模型进行日志总结。**注意：启用后插件会将日志内容和Maisaka推理输出发送到你配置的外部 LLM API（默认 DeepSeek），请确保你接受这一行为。**
- **API密钥**：DeepSeek API密钥（WebUI中显示为 `********`，可手动点击显示明文）。
- **API地址**：OpenAI兼容的API端点（默认 `https://api.deepseek.com`）。
- **模型名称**：要调用的模型名称（默认 `deepseek-chat`）。
- **温度**：控制生成的随机性（0-2，步进0.1，默认0.7）。

## 命令

| 命令 | 说明 |
|------|------|
| `发生什么了{bot_name}` | 查看最近日志总结 + Maisaka 推理（含 LLM 总结，不受 WebUI 限制） |
| `是不是哪里出问题了` | 同上 |
| `查你后台` | 提取当前聊天流的 Maisaka 后台推理输出（所有平台可用，不经过 LLM 总结） |
| `/whatsmailog` | 手动触发日志查看（受"命令仅限WebUI"开关控制） |
| `/check_backend` | 提取当前聊天流的 Maisaka 后台输出（受"命令仅限WebUI"开关控制） |
| `/llmtest` | 测试 LLM 提供商连接是否正常 |
| `/logfiles` | **调试**：列出日志目录中所有 log.jsonl 文件、大小及来源进程 |
| `/logpath` | 查看当前配置的日志文件路径和状态 |

> **注意**：所有 `/xx` 命令均受"命令仅限WebUI"设置影响。开启后，非 WebUI 平台触发命令时插件仅记录日志，不会发送任何消息。触发词（如"发生什么了"、"查你后台"）不受此限制。

## 工作原理

### 日志源

MaiBot 使用两种日志系统：

| 来源 | 路径 | 格式 | 内容 |
|------|------|------|------|
| 系统日志 | `logs/app_*.log.jsonl` | JSONL（每行一个 JSON） | 所有进程的系统日志 |
| Maisaka 推理 | `logs/maisaka_prompt/{kind}/{chat}/` | 结构化 JSON 文件 | Planner 分析、Replyer 回复等 |

> **重要**：MaiBot 每个进程（Host + 多个 Runner）各自创建独立的 `app_*.log.jsonl` 文件。插件会读取所有近期文件以确保不遗漏日志。

### 处理流程

1. **触发时机**：当用户发送配置的触发词或命令时触发。
2. **日志获取**：
   - 系统日志：读取 `logs/` 目录下所有近期的 `app_*.log.jsonl` 文件，按时间过滤。
   - Maisaka 输出：从 `logs/maisaka_prompt/planner/` 和 `replyer/` 读取最新的推理 JSON 文件，优先匹配当前聊天流目录。
3. **LLM总结**：若启用了 LLM，将系统日志和 Maisaka 推理输出一起发送给 LLM 进行综合总结（150字以内）。**注意：启用后这些内容会发送到你配置的外部 LLM API。**
4. **错误检测**：如果日志中包含 `ERROR` 或 `CRITICAL` 字样，回复中增加 `⚠️` 告警标记。
5. **日志片段**：在消息末尾附加"最新日志片段"（最近3行），帮助快速了解当前状态。
6. **查你后台**：单独提取当前聊天流的 Maisaka 推理输出原文，不经过 LLM 总结。
7. **时间戳**：在报告末尾添加生成时间，方便追溯。
8. **日志记录**：所有操作均通过 `self.ctx.logger` 记录详细日志到主进程，方便调试。

## 注意事项

- 请确保 `日志文件路径` 正确指向 MaiBot 的日志目录（默认 `logs/`）。
- 插件需要同时访问 `logs/app_*.log.jsonl` 和 `logs/maisaka_prompt/` 目录。
- `app_*.log.jsonl` 中可能不包含 Maisaka 推理输出（取决于 MaiBot 版本），但 `maisaka_prompt/` 始终包含完整的结构化推理记录。
- API密钥在WebUI中默认脱敏显示，可手动切换明文，建议保管好密钥。
- 如果LLM调用失败，插件会自动回退到不带总结的纯日志片段展示。
- 启用 LLM 生成功能会将日志内容和 Maisaka 推理输出发送至你所配置的外部 API（如 DeepSeek），请确认你接受这一隐私风险。
